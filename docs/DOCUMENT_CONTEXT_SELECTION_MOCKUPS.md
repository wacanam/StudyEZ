```
N. Last document checkbox
```

## Design Decisions & Rationale

### Why Collapsible?
- **Space efficiency**: Doesn't take up permanent screen space
- **Focus**: Allows users to focus on query input when not needed
- **Progressive disclosure**: Advanced feature doesn't overwhelm beginners

### Why Show Count in Collapsed State?
- **Visibility**: Users can see their selection without expanding
- **Context**: Provides quick reference before querying
- **Confidence**: Users know the feature is working

### Why Search Bar?
- **Scalability**: Essential for users with many documents
- **Usability**: Quick way to find specific files
- **Expected pattern**: Common in selection interfaces

### Why Select All / Clear Buttons?
- **Efficiency**: Common batch operations
- **Time-saving**: Faster than individual selection
- **User expectation**: Standard in multi-select interfaces

### Why Warning for No Selection?
- **Transparency**: Makes it clear all documents will be searched
- **Education**: Helps users understand the default behavior
- **Encouragement**: Prompts users to try the selection feature
